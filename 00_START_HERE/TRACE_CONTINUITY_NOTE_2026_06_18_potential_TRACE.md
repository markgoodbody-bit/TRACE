## Core discovery

```trace
potential_TRACE :=
  any_domain_may_contain_traceable_patterns
  if:
    change
    + constraint
    + consequence
    + memory
    + correction_or_drift

active_TRACE :=
  TRACE_becomes_useful
  only_when_it_improves:
    visibility
    OR timing
    OR correction
    OR hidden_burden_detection
    OR next_action

false_TRACE :=
  lens_forced_onto_material
  + no_added_value
  + decorative_depth
```

Plain English: **everything is potentially TRACE, but TRACE is only active when it helps.**

This protects the work from two opposite failures:

1. Narrowing too far, so TRACE cannot cross domains.
2. Totalising the lens, so every case is forced into TRACE whether or not it adds value.



Potential contribution, if any, is not the primitives. It is the arrangement around harm, time, correction, witness, and answerability.

## Activation test

Use TRACE when it helps answer:

```trace
activation_test :=
  what_is_changing?
  + who_or_what_is_affected?
  + what_constraint_or_cost_is_active?
  + what_information_or_memory_carrier_matters?
  + what_is_hardening?
  + what_correction_channel_exists?
  + can_it_arrive_in_time?
  + what_action_follows?
  + what_would_a_no_TRACE_reasoner_see?
```

If a competent no-TRACE reasoner sees the same structure, timing, and next action, mark the use as `COMPRESSION_ONLY` or `NO_ADDED_VALUE`.
