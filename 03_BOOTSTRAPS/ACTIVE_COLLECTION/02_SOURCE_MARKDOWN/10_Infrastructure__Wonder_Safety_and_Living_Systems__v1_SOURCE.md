## Missing Rosetta-Core Material

The current Rosetta layer does not yet carry infrastructure strongly enough.

Infrastructure should not be treated as a noun: a bridge, house, telescope, railway, server, model, institution, or city.

Infrastructure is a maintained constraint system over time.

```trace
infrastructure :=
  material_form
  + operating_method
  + record
  + maintenance
  + safety_default
  + authority_map
  + hidden_basement
  + future_space_effect
```

The bootstrap asks:

```trace
infrastructure_question :=
  does_the_system_make_future_space
  while_preserving:
    safety
    + record
    + correction
    + repair
    + honest_provenance?
```

## Case 1 — Cragside / Wonder With a Basement

Cragside shows the future arriving early for the rich: hydroelectricity, central heating, telephone links, hydraulic systems, powered kitchen machinery, electric lighting, and domestic automation.

But the wonder is not clean merely because it is impressive.

```trace
Cragside_lesson :=
  the_future_can_arrive_locally
  before_the_correction_structure_arrives_globally
```

The visible system is luxury. The hidden system includes servants, child risk, electrified mercury, coal, industrial extraction, and arms-capital provenance.

```trace
Cragside_basements :=
  servant_labour
  + child_risk
  + electrified_mercury
  + coal
  + imperial_arms_capital
  + distant_harm_pipeline
```

The lamp system is the cleanest object lesson:

```trace
lamp_system :=
  visible_output: light
  hidden_method:
    ten_year_old_handler
    + insulated_gloves
    + electrified_mercury
    + no_domestic_switch_standard
```

The future is real. The method is not clean.

```trace
technical_success != ethical_completion
```

### K-Gate

```trace
K_Cragside_lamps :=
  Witness: partial
  Record: weak
  Authority: owner/engineer concentrated
  Time: immediate toxicity/electrocution risk
  Enforcement: immature/absent
  Repair: after_the_fact_only
```

wonder_basement :=

This is not a new operator. It is a translation surface for basement routing and method laundering.

## Case 2 — Railway Signalling / Safety Embedded Before Hazard Entry

Rail signalling is a positive case for the narrowed TRACE spine.

The key lesson is not that signallers are careful. It is that the system makes many dangerous states physically or logically unavailable.

```trace
rail_safety_law :=
  do_not_make_safety_depend_on_last_second_judgment
  when_you_can_make_dangerous_states_unavailable
```

```trace
rail_signalling :=
  make_wrong_order_impossible
  + treat_unknown_as_danger
  + require repeated_permission
  + bind_communication_to_machinery
  + investigate_boundary_violations
  + train_for_abnormal_cases
```

### K-Gate

```trace
K_rail :=
  Witness: track_circuits + windows + diagrams + tail_lamps
  Record: signal_box_book + block_instruments + repeated_bell_codes
  Authority: signaller_can_clear_or_block_route
  Time: clearance_must_precede_block_entry
  Enforcement: mechanical/electrical_interlocks
  Repair: stop + investigate + reset + train + review_SPADs
```

### Core Safety Default

```trace
unknown != clear
unknown := danger
```

Track circuits, block indicators, bell codes, tail lamps, interlocks, and repeat-back communication all push in the same direction: uncertainty blocks permission.

### Absolute Block

```trace
absolute_block :=
  one_train_per_signalled_block
  + no_following_train_without_protected_stop
```

Safety is not local emptiness. Safety is stoppable structure.

```trace
safe_route != empty_segment

safe_route :=
  empty_enough
  + signal_boundary
  + stopping_distance
  + authority_to_hold_next_train
```

### Near-Miss Discipline

```trace
SPAD := signal_passed_at_danger

if no_crash:
  still_investigate

near_miss != no_event
```

A good safety system does not let a good outcome clean a boundary violation.

```trace
outcome_ok
  does_not_clean
  safety_boundary_violation
```

### Rosetta Delta

Rosetta needs to say more clearly:

```trace
embedded_safety :=
  correction_moved_upstream
  before_hazard_entry
```

This supports Correction Before Hardening and K-Gate. It does not add an operator.

### Must Not Claim

```trace
rail_case_must_not_claim :=
  old_systems_are_always_better
  OR modernisation_is_bad
  OR human_operators_are_enough
  OR fail_safe_design_eliminates_all_risk
```

## Case 3 — Jodrell Bank / Living Scientific Infrastructure

Jodrell Bank shows that old infrastructure can remain scientifically alive if the right layers are maintained, repaired, upgraded, and recalibrated.

```trace
living_infrastructure :=
  stable_core
  + replaceable_layers
  + repair_capacity
  + calibrated_record
  + future_question_capacity
```

```trace
old != obsolete
heritage != dead
continuity != component_freeze
```

The Lovell Telescope remains useful because the core scientific form still works, while the surface, electronics, control systems, detectors, focal packages, and networked correlation have changed.

```trace
Lovell_Telescope :=
  old_core_structure
  + maintained_surface
  + upgraded_control
  + replaceable_detectors
  + networked_correlation
  + live_science
```

### K-Gate

```trace
K_Jodrell :=
  Witness: radio_signals + calibrators + pulsars + asteroids
  Record: raw_signals + correlated_data + timing_records + archives
  Authority: operators + engineers + observatory_governance
  Time: observing_windows + maintenance_cycles + infrastructure_lifespan
  Enforcement: control_procedures + radio_quiet_rules + access_controls
  Repair: painting + wheel_changes + surface_replacement + electronics_upgrades + detector_swaps
```

### Scientific Witness

```trace
scientific_witness :=
  signal_capture
  + calibration
  + correlation
  + error_management
  + archive
```

The instrument is not valuable merely because it inspires awe. It remains valuable because it can still make the unknown answerable through calibrated record.

```trace
awe != method
heritage != live_science
```

### Heritage Tension

```trace
heritage_status :=
  protects_structure
  but_may_constrain_upgrade

preservation_guard :=
  preserve_if_supports_memory_or_function
  but_do_not_preserve_so_rigidly
  that_living_system_cannot_repair
```

### Rosetta Delta

Rosetta needs to distinguish static preservation from living continuity.

```trace
living_continuity :=
  stable_identity
  + replaceable_parts
  + maintained_function
  + upgrade_path
```

This supports Record Trust, Repair, and Option-Space Restoration. It does not add an operator.
