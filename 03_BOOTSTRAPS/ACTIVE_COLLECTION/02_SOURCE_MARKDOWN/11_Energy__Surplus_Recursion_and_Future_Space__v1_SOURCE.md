## Why Energy Belongs Here

TRACE already has state, transition, constraint, future-space, method, time, and correction. But it does not yet say clearly enough that every transition requires capacity.

Energy is the most basic form of capacity-to-change-state.

```trace
energy :=
  capacity_to_change_state
```

For TRACE, the relevant question is not only whether energy exists, but how it is routed, governed, converted, constrained, stored, wasted, weaponised, repaired, or hidden.

```trace
ethical_energy :=
  capacity_to_change_state
  while_preserving:
    correction
    + repair
    + exit
    + future_space
    + non_hidden_cost
```

Energy is not automatically good. Energy widens action-space, but it can also widen domination-space.

```trace
surplus_energy :=
  more_possible_action
  + more_possible_repair
  + more_possible_acceleration
  + more_possible_extraction
  + more_possible_harm
```

## Energy as Future-Space

Energy increases the reachable region of future-space.

```trace
Xi_t := accessible_future_option_space

energy_delta > 0
  can_expand Xi_{t+k}
```

But the sign is not automatic.

```trace
energy_use_valid iff:
  Xi_subjects expands_or_preserved
  + K remains_live
  + method_floor_preserved
  + basement_cost_not_hidden
```

If energy expands one actor's future-space by collapsing another's, TRACE should not call that simple progress.

```trace
false_progress :=
  Xi_center expands
  while Xi_edge collapses
```

## Surplus and Repair

Surplus is not luxury by default. Surplus can be a repair buffer.

```trace
surplus_good_when :=
  absorbs_shock
  + speeds_correction
  + preserves_redundancy
  + funds_maintenance
  + protects_exit
  + reduces_downward_burden_routing
```

Surplus becomes dangerous when it decouples power from answerability.

```trace
surplus_bad_when :=
  accelerates_harm
  + hides_cost
  + captures_authority
  + outbids_repair
  + converts_need_into_dependency
  + routes_exhaustion_downward
```

Energy is therefore a K-gate concern.

```trace
K_energy :=
```

## Energy and Time

Energy changes timing.

```trace
energy_can_reduce_tau_correct
energy_can_reduce_tau_harden
```

This is the knife-edge.

More energy can speed repair. It can also speed harm hardening.

```trace
repair_use :=
  energy -> shorter_tau_correct

harm_use :=
  energy -> shorter_tau_harden
```

TRACE must ask which clock energy is accelerating.

```trace
energy_clock_question :=
  does_this_energy_use_make_correction_faster
  or_hardening_faster?
```

## Mandelbrot / Recursion Role

Mandelbrot does not prove TRACE.

Mandelbrot is useful as a structural analogy: simple local rules iterated under boundary conditions can generate complex large-scale structure.

```trace
Mandelbrot_role :=
  analogy_for_recursive_structure
  not:
    proof
    operator
    metaphysics
    validation
```

Energy belongs in that recursion because repeated small energy flows build large structures.

```trace
recursive_energy_pattern :=
  local_energy_rule
  + feedback
  + boundary
  + time_iteration
  -> large_scale_structure
```

The important TRACE lesson is not fractal aesthetics. It is scale continuity.

```trace
same_pattern_can_recur_at:
  body
  + household
  + institution
  + city
  + grid
  + civilisation
  + AI_system
```

At each scale:

```trace
energy_route :=
  capacity
  + constraint
  + feedback
  + waste
  + repair
  + hidden_cost
```

A civilisation is not good because it has more energy. It is better only if it routes energy through constraints that preserve future-space and correction.

## Energy, Agency, and Free Will Boundary

This file does not settle free will.

Free will remains candidate-annex material.

But energy clarifies one boundary: agency requires usable option-space, and usable option-space requires capacity.

```trace

  faster_deployment
  + larger_action_radius
  + harder_competition_pressure
  + datacenter_lock_in
```

A high-energy AI system with low K is structurally dangerous.

```trace

  risk := fast_uncorrectable_state_change
```

```trace
answerable_energy :=

  + bounded
```
