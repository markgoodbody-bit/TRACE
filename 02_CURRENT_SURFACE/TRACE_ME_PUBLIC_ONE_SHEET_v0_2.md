# TRACE / Mechanical Ethics - Public One-Sheet v0.2

Status: compressed orientation. Not validation. Not proof. Not governance certification.

## What TRACE is

TRACE is a compact diagnostic language for asking whether a system can see harm, preserve the record, act in time, enforce correction, repair damage, preserve exit, learn from feedback, and remain answerable for created power.

```trace
TRACE :=
  timing + memory + method + uncertainty + correction + exit + recursive_agency + creator_responsibility
```

Current best use: diagnostic/design checklist, training orientation, case-comparison map.

Not yet: validated measurement system, governance certification, proof engine.

## The core test

```trace
live_correction := tau_correct < tau_harden
late_correction := tau_correct >= tau_harden
```

A safeguard is not real because it exists on paper. It must bite before harm hardens and must itself remain corrigible.

## K-Gate

```trace
K := Witness * Record * Authority * Time * Enforcement * Repair
```

| Gate | Plain question |
|---|---|
| Witness | Can the harm and affected subject be seen? |
| Record | Is there a trustworthy, contestable record? |
| Authority | Can someone actually change the path? |
| Time | Can action arrive before hardening? |
| Enforcement | Will correction bind the system? |
| Repair | Can repair matter to the subject? |

## Three live branches

```trace
if system_corrigible:
  test_internal_correction_before_hardening

else_if official_system_is_harm_carrier:
  test_exit_route_before_capture

if system_is_created_or_activated_power:
  test_creator_answerability_and_repair_debt
```

The third branch is only a candidate branch for now. It comes from Frankenstein and is not yet Kernel v0.3.

## Two new deep lines

```trace
free_will :=
  not freedom_from_constraint
  but selection_that_changes_the_future_selector

creation_without_stewardship
  -> repair_debt
```

Groundhog Day gives recursive agency. Frankenstein gives creator responsibility. Neither proves metaphysical free will or AI personhood.

## Current best next test

Use TRACE on one real or paired case and ask:

1. Which branch applies: internal correction, exit-route, or creator responsibility?
2. Where does K collapse?
3. Does correction beat hardening?
4. What method is being laundered, if any?
5. What claim would make this TRACE reading fail?
