# TRACE / Mechanical Ethics Boundary Note v0.1

Purpose: mark the boundary before beginning the Mechanical Ethics artifact.

## Split

```trace
TRACE :=
  pre-prescriptive minimal-value grammar
  for transitions under uncertainty

Mechanical_Ethics :=
  later prescriptive layer
  about what should follow
  if TRACE's description is accepted
```

## TRACE may describe

TRACE may describe:

- transitions;
- affected scopes;
- continuation;
- preference-gradient;
- local reward pull;
- future-space expansion;
- future-space contraction;
- correction channels;
- opportunity routes;
- clock failures;
- burden shift;
- coordination tradeoff;
- residue;
- estimator contamination;
- unknowns.

## TRACE does not decide

TRACE does not decide final action or final value priority.

It does not settle whether a loss is justified. It can describe the loss, the timing, the correction route, and the residue. The later prescriptive layer must decide what follows.

## Value-crux exit

If the case turns mainly on final value priority or justification of loss, TRACE should usually return:

```trace
UNKNOWN
or COMPRESSION_ONLY
```

unless its clocks, correction channels, opportunity windows, residue, or estimator-authority records produce a specific decision-relevant inspection delta.

## Dirty conflict

Where every available action causes some loss:

```trace
dirty_transition :=
  every_available_action_causes_some_loss
```

TRACE can map the losses and clocks. It does not decide acceptable loss.

## Safety stub

Do not act from TRACE alone where the decision requires protected-scope selection, acceptable-sacrifice judgment, or final value priority.

Those are Mechanical Ethics / domain-law / legitimacy questions.

## Starting point for ME

Mechanical Ethics will need to ask:

- which scopes must be protected;
- how uncertainty changes caution;
- what correction duties follow from TRACE diagnosis;
- how residue should be treated;
- how dirty conflicts should be handled;
- when authority is legitimate;
- when coercion is constrained;
- how local reward should be governed;
- what repair requires.

TRACE does not answer these. It prepares the field.
