## Core claim

Ecology makes hidden bills literal.

```trace
ecological_hidden_bill :=
  extract_now
  + defer_waste_or_damage
  + obscure_downstream_path
  + future_system_pays
```

Plain version: a system can look profitable, clean, efficient, or stable at the point of action because cost has been routed into soil, water, air, bodies, future repair capacity, or later generations.

## Why ecology matters to TRACE

TRACE already tracks harm that is delayed, displaced, hidden, or routed onto weaker subjects.

Ecology adds the material version:

```trace
ecological_routing :=
  local_benefit
  -> distributed_damage
  -> delayed_return
  -> hard_to_assign_responsibility
```

This connects to:

```trace
old_TRACE_strengths :=
  hidden_bill
  + debt_clock
  + correction_before_hardening
  + carrier_realism
  + future_space_collapse
```

Ecology gives those structures a substrate: energy, waste, regeneration, carrying capacity, and delayed feedback.

```trace
hidden_bill_forms :=
```

```trace
routing_pattern :=

```

## Delayed feedback

Ecological systems often do not fail immediately.

```trace
delayed_feedback :=
  damage_accumulates
  + system_buffers
  + visible_failure_lags
  + correction_window_closes
```

This is the ecological form of “before it hardens.”

```trace
eco_hardening :=
  damage_crosses_threshold
  -> repair_cost_jumps
  -> restoration_becomes_partial_or_impossible
```

Examples of the structure, not claims about specific cases:

```trace
examples_as_patterns :=
  overfishing_before_stock_collapse
  + runoff_before_dead_zone
  + emissions_before_climate_feedback
  + habitat_loss_before_species_absence
  + stress_before_body_breakdown
```

## Carrying capacity

```trace
carrying_capacity :=
  regeneration_rate
  + absorption_capacity
  + system_load
  + time
```

Overshoot occurs when load exceeds regeneration or absorption for long enough that the system’s future capacity is reduced.

```trace
overshoot :=
  use_rate > repair_or_regeneration_rate
  -> future_capacity_loss
```

TRACE translation:

```trace
overshoot_as_hidden_bill :=
  present_use
  + undercounted_regeneration_time
  -> future_subject_pays
```

## Ecological debt clock

```trace
ecological_debt_clock :=
  damage_rate
  - repair_or_regeneration_rate
  -> accumulated_debt
```

This is not a precise equation here. It is a diagnostic orientation.

```trace
if damage_rate > repair_rate:
  hidden_bill_grows
```

The debt clock becomes dangerous when the system treats buffers as proof of safety.

```trace
buffer_misread :=
  system_absorbs_damage_for_now
  -> actor_claims_no_harm
  -> correction_delayed
```

Demoter:

```trace
debt_clock_demoter :=
  if_note_claims_exact_measure_without_data
  then_overclaim
```

## Repair lag

```trace
repair_lag :=
  time_between_damage_and_recovery
  + resource_needed
  + subject_or_ecology_capacity
  + irreversibility_risk
```

Ecological repair often requires time beyond human reporting cycles, election cycles, product cycles, or quarterly accounting.

```trace
cycle_mismatch :=
  fast_extraction_cycle
  + slow_repair_cycle
  -> structural_hidden_bill
```

Plain version: if benefit is fast and repair is slow, accounting will tend to lie unless it explicitly includes repair time.

## Distributed victims

Ecological hidden bills often lack a single easily visible victim.

```trace
distributed_victim_problem :=
  many_small_burdens
  + delayed_body_effects
  + nonhuman_subjects
  + future_subjects
  + weak_claim_channels
```

This makes denial easier.

```trace
denial_route :=
  no_single_visible_victim
  -> no_clear_claimant
  -> no_repair_carrier
```

TRACE correction:

```trace
ecological_witness_need :=
  measurement
  + preserved_records
  + independent monitoring
  + future-facing accounting
```

## Benefit trace and harm trace

Older TRACE enforcement-carrier work emphasised that harm trace needs benefit trace.

Ecology makes this non-optional.

```trace
benefit_trace_ecology :=
  who_gained?
  + what_was_extracted?
  + what_cost_was_deferred?
  + who_lacks_power_to_refuse?
  + who_pays_later?
```

Hidden bills are not only harms. They are often harms paired with benefits routed elsewhere.

```trace
hidden_bill_full :=
  benefit_here
  + cost_elsewhere
  + time_gap
  + responsibility_gap
```
