# TRACE Predation / Symbiosis Contrast v0.1

Status: support note inside the Life / Biology / Ecology Atlas. Not canon. Not validation. Not biology textbook. Not moral permission derived from nature. This note prevents `predation` from becoming a lazy universal label by contrasting predation, parasitism, mutualism, commensalism, stewardship, dependence, and moral extraction.

## Why this note exists

TRACE often uses predation as a strong pattern: fast extraction, weak correction, burden routed onto a vulnerable subject. That remains useful.

But overuse would damage the framework.

```trace
predation_overuse_risk :=
  every_asymmetry_called_predation
  -> loss_of_precision
  -> moral_theatre
```

Biology gives multiple relation forms. Some are harmful, some mutual, some neutral, some mixed, and some shift over time.

```trace
relation_forms :=
  predation
  + parasitism
  + mutualism
  + commensalism
  + competition
  + stewardship
  + dependency
  + care
```

## Core distinction

```trace
predation :=
  one_life_consumes_another
  + target_future_space_ends_or_collapses
```

```trace
symbiosis :=
  different_life_forms_share_relation
  + dependency_or_benefit_structure
```

Plain version: not every asymmetry is predation. Some asymmetry is care. Some dependency is mutual. Some help is disguised extraction. The work is to distinguish them.

## Natural predation is not moral permission

```trace
natural_predation != moral_permission
```

Biological predators eat because they must. Human and institutional predation often uses “nature” language to excuse avoidable extraction.

```trace
naturalisation_error :=
  nature_contains_predation
  -> actor_claims_permission_to_extract
```

Demoter:

```trace
if_nature_used_as_ethics_shortcut:
  mapping_failed
```

## Predation pattern in TRACE

```trace
predation_pattern :=
  fast_extraction
  + target_cannot_correct_in_time
  + benefit_to_predator
  + future_space_loss_for_target
```

Social/institutional predation is strongest when:

```trace
moral_predation_indicators :=
  avoidable_extraction
  + asymmetric_power
  + weak_exit
  + weak_contest
  + benefit_trace_hidden
  + harm_trace_disputed
```

Use predation only when these features are materially present.

## Parasitism

```trace
parasitism :=
  host_dependency
  + extraction
  + host_kept_alive_enough
  + concealed_or_managed_damage
```

Parasitism differs from simple predation because the extractor may need the host functional.

```trace
parasitic_system :=
  keep_host_viable
  + route_burden_to_host
  + extract_repeatedly
```

Institutional analogue:

```trace
institutional_parasitism :=
  subject_or_worker_or_ecology_needed
  + burden_loaded_onto_them
  + system_claims_mutuality
  + exit_hard
```

Demoter:

```trace
parasitism_demoter :=
  if_host_benefits_too
  then_check_for_mutualism_or_mixed_relation
```

## Mutualism

```trace
mutualism :=
  both_parties_gain_capacity
  + relation_stabilises_future_space
```

Mutualism is not sameness or equality. It can be asymmetric and still genuinely mutual if both parties gain and neither is trapped in hidden damage.

```trace
healthy_mutualism :=
  benefit_both_sides
  + burden_visible
  + exit_or_adjustment_possible
  + no_hidden_degradation
```

Examples as structure, not scientific proof claims:

```trace
mutualism_patterns :=
  pollination
  + microbiome_support
  + human_dog_relation_when_welfare_respected
  + good_tool_use
  + cooperative_work_under_fair_terms
```

Demoter:

```trace
mutualism_demoter :=
  if_one_side_cannot_exit
  OR hidden_burden_accumulates
  OR benefit_trace_one_sided
  then_mutualism_claim_degrades
```

## Commensalism

```trace
commensalism :=
  one_party_benefits
  + other_party_not_materially_harmed_or_helped
```

This category matters because not every benefit extracted from proximity is exploitation.

```trace
commensalism_guard :=
  check_material_harm
  + check_burden
  + check_hidden_cost
```

Demoter:

```trace
commensalism_demoter :=
  if_cost_is_hidden_not_absent
  then_reclassify
```

## Competition

```trace
competition :=
  shared_limited_resource
  + multiple_claimants
  + selection_pressure
```

Competition is not automatically predation. It can be bounded, regenerative, and fair enough, or it can become destructive.

```trace
bounded_competition :=
  fair_rules
  + non-destruction_of_loser
  + regeneration_possible
  + contest_path
```

```trace
destructive_competition :=
  winner_gain
  + loser_future_space_collapse
  + no_repair_or_reentry
```

Demoter:

```trace
competition_demoter :=
  if_loser_destroyed_or_excluded_permanently
  then_competition_may_have_become_predatory
```

## Stewardship

```trace
stewardship :=
  power_over_living_or_ecological_system
  + duty_to_preserve_future_capacity
  + restraint
  + repair_obligation
```

Stewardship is a positive asymmetry. It is power that binds itself to future-space preservation.

```trace
healthy_stewardship :=
  asymmetry_acknowledged
  + extraction_limited
  + repair_funded
  + future_subjects_counted
```

Demoter:

```trace
stewardship_demoter :=
  if_power_calls_itself_care
  but:
    extracts_without_repair
    + hides_benefit
    + blocks_exit_or_contest
  then_stewardship_claim_fails
```

## Dependency and care

```trace
dependency :=
  one_subject_needs_another
  + asymmetric_capacity
```

Dependency is not automatically weakness, failure, or prey-status.

```trace
care_relation :=
  dependency
  + responsibility
  + attention
  + subject_welfare_central
```

This links directly to Maisie.

```trace
care_not_predation :=
  asymmetry
  + dependent_subject
  + burden_taken_by_carer
  + subject_future_space_preserved
```

Demoter:

```trace
care_demoter :=
  if_dependent_subject_used_for_carer_need_only
  then_care_claim_degrades
```

## Extraction

```trace
extraction :=
  take_value
  + route_benefit_to_actor
  + impose_cost_on_source_or_context
```

Extraction is not always wrong. Living requires energy and material exchange. The ethical question turns on necessity, proportionality, renewal, burden, consent/standing, and repair.

```trace
responsible_extraction :=
  necessity_or_legitimate_use
  + bounded_amount
  + regeneration_or_repair
  + burden_visible
  + affected_subjects_counted
```

```trace
exploitative_extraction :=
  avoidable_take
  + hidden_burden
  + weak_subject_claim
  + no_repair
```

## Relation drift

Relations can change type.

```trace
relation_drift :=
  mutualism
  -> commensalism
  -> parasitism
  -> predation
```

or the other direction through repair.

```trace
relation_repair :=
  hidden_burden_seen
  + extraction_reduced
  + repair_funded
  + exit_or_contest_restored
```

TRACE should treat relation-type as dynamic, not fixed.

## Diagnostic questions

```trace
relation_diagnostic :=
  who_benefits?
  + who_pays?
  + is the burden visible?
  + can the weaker party exit_or_contest?
  + is repair_or_regeneration funded?
  + does one side's future_space_expand_or_collapse?
  + is the relation necessary_or_avoidable?
  + does the relation drift_under_pressure?
```

## Contrast table

| Relation form | Basic structure | Key TRACE test |
|---|---|---|
| Predation | One consumes another; target future-space collapses. | Is extraction fast, avoidable, and correction-impossible for target? |
| Parasitism | One extracts while keeping host viable enough. | Is the host still functioning but degrading under hidden burden? |
| Mutualism | Both gain capacity/future-space. | Are burdens visible and adjustable, with no hidden degradation? |
| Commensalism | One benefits; other not materially affected. | Is cost truly absent, or merely hidden/displaced? |
| Competition | Multiple claimants under scarcity. | Are rules bounded, and can losers recover/re-enter? |
| Stewardship | Asymmetric power bound to future preservation. | Is power funding repair and counting future subjects? |
| Care | Dependency met by responsibility and attention. | Is dependent subject welfare central, not carer need alone? |
| Exploitation | Value taken with hidden or imposed burden. | Who benefits now, who pays later, and who cannot contest? |

## Anti-overuse guard

```trace
anti_overuse_guard :=
  do_not_call_predation_when:
    bounded_competition
    OR genuine_mutualism
    OR responsible_stewardship
    OR care_relation
    OR necessary_life_exchange
```

Use stronger labels only when the structure earns them.
