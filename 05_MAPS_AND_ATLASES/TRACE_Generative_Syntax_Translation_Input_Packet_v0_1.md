## 1. Domain name

```trace
domain_name := Generative_Syntax_Minimalist_Program
```

Plain version:

This domain studies the structure of human language through formal operations and constraints that generate syntactic objects and derivations.

## 2. Native object space

The native object space is not a physical state space, a topology space, a quantum space, or a macro model space. It is a space of syntactic objects, lexical items, features, and derivational structures.

```trace
native_syntax_object_space :=
  lexical_items
  + features
  + syntactic_objects
  + derivation_steps
  + hierarchical_structures
  + dependency_relations
  + interface_representations
```

Candidate native object for first probe:

```trace
native_object_or_case :=
  simple_Merge_derivation_profile
  under:
    simplified_internal_translation_probe
```

Boundary:

This is not a full Minimalist grammar. It is an input packet for testing whether TRACE can preserve the native shape of a syntactic derivation without turning syntax into word order, meaning, or generic combination.

## 6. Critical distinctions

The translation must preserve these distinctions:

```trace
critical_distinctions :=
  Merge_vs_generic_combination
  + syntactic_object_vs_surface_string
  + hierarchy_vs_linear_order
  + derivation_vs_sentence_meaning
  + feature_dependency_vs_semantic_association
  + Internal_Merge_vs_arbitrary_reordering
  + Agree_vs_agreement_as_vague_similarity
  + grammaticality_judgment_vs_truth_value
  + interface_readout_vs_underlying_derivation
```

Plain version:

A syntax translation fails if it turns formal syntactic derivation into ordinary word order, meaning, or loose association.
