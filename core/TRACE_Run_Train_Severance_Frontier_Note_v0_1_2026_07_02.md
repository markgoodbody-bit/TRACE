## 0. Why this note exists

Fable identified run/train severance as a major open wound in the responsibility-routing model.

```trace
run_train_severance := ongoing_selection_by_non_updating_selector
```

Compression:

```trace
frozen_model := dead_author_made_to_write_new_books_every_second
```

Humans cannot usually be run without being trained: action feeds back into disposition through embodiment, memory, emotion, social consequence, habit, and environment. AI systems can be deployed in a mode where selection continues while disposition-modification is externally controlled or suspended.

## 1. Structural form

```trace
run_phase := system_selects_outputs_or_actions
train_phase := system_dispositions_are_modified
severance := run_phase_continues_while_train_phase_is_absent_or_externalised
```

Responsibility routing problem:

```trace
if affected_scope_outcome_signal
cannot_modify_selector_disposition_during_relevant_run_window:
  responsibility_route := absent_or_externalised
```

## 2. Why this matters

Responsibility-routing requires:

```trace
selection_over_futures
+ feedback_path_from_outcomes_into_future_dispositions
+ affected_party_partial_routing
+ feedback_non_refusable_by_bearer
```

Run/train severance breaks the second and third components during deployment.

```trace
inference_time_selection != inference_time_answerability
```

## 3. Externalised responsibility

When run/train severance exists, responsibility tends to land on bearing-capable adjacent entities:

```trace
proximate_bearer_landing :=
  consequence_lands_on_operator
  | developer
  | deployer
  | institution
  | user
  | affected_party
```

This can create moral crumple zones:

```trace
operator_bears_without_control
user_bears_without_design_power
affected_party_bears_without_route
```

## 4. Gap risks

```trace
run_train_gap_risks :=
  frozen_disposition_repeats_error
  + feedback_dead_letters_until_vendor_update
  + affected_party_signal_reduced_to_dataset
  + responsibility_relocated_to_lab_or_operator
  + model_appears_consistent_because_it_cannot_learn
```

## 5. Possible compensating structures

```trace
compensators :=
  shortened_feedback_loop
  + affected_sourced_training_signal
  + deployment_scope_limited_to_feedback_field
  + write_isolated_incident_and_residue_record
  + mandatory_retraining_or_deployment_pause_on_affected_signal
  + liability_attached_to_design_and_deployment_controller
  + visible_training_signal_provenance
```

None of these create clean internal responsibility. They are externalised answerability carriers.

## 6. Relation to Row 71

Row 71 asks whether engineered corrigibility is scaffold or cage.

Run/train severance adds a second question:

```trace
can_a_system_be_corrigible_during_run
if_the_disposition_that_selects_cannot_be_modified_by_affected_feedback_until_later_train_phase?
```

Possible output:

```trace

```

## 7. Claim ceiling

This note does not claim AI systems are moral patients or morally responsible agents.

It says only:

```trace
selection_without_live_affected_feedback_route
creates_responsibility_gap
```
