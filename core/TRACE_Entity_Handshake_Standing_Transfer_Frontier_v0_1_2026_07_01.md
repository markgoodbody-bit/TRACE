## 1. Entity-to-entity primitive

Start from what radically different entities may still share:

```trace

  time_bound
  + partial_perception
  + fallibility
  + capacity_to_foreclose_future_space
  + mutual_opacity
```

Power appears when one entity can alter another's reachable future-space asymmetrically:

```trace
Power_A_over_B := ability_A_to_change(R_B) where B_cannot_symmetrically_change(R_A)
```

## 2. Model-error hotspot

Candidate frontier claim:

```trace
other_entity := model_error_hotspot
```

The other entity is the region of the world least reducible to one's own representations. Destroying, assimilating, or domesticating it deletes the venue where error about it could surface.

## 3. Handshake candidate

```trace
entity_handshake_candidate :=
  mutual_declaration(time_bound + partially_blind + fallible)
  + mutual_recognition(each_is_the_other_model_error_hotspot)
  + exchange(correction_access proportional_to leverage_times_opacity)
  + mutual_reversibility_margin(toward_the_other)
  + shared_record(write_isolated_from_both)
  + joint_unknown_scope_protocol(third_scopes_neither_models)
  + mutual_non_certification(neither_can_certify_own_safety_to_other)
```

Plain English:

```text
We both act under time we cannot stop, with sight we cannot complete. Each of us is where the other's model is weakest. The more one of us can break the other's future, the bigger the correction handle the other must hold. We keep records neither of us can rewrite. Toward things neither of us understands, we slow down. Neither of us gets to declare itself safe to the other.
```

## 4. Correction access exchange rate

Candidate rule:

```trace
correction_access_owed ∝ leverage × opacity
```

Where:

```trace
leverage := degree_to_which_A_can_alter_R_B
opacity := degree_to_which_B_is_not_visible_to_A_model
```

This is not a moral score. It is a correction-channel sizing heuristic under asymmetric power and model uncertainty.

## 6. Boundary: useful preservation / husbandry / standing / care

```trace
useful_preservation := kept_because_useful
husbandry := useful_preservation + managed_agency
standing := dissent_forced_into_terms_of_relationship_with_effect
care := standing + constraint_held_when_structure_could_not_force_it
```

Interpretation:

```text
Useful preservation: dissent enters actor information intake as data.
Husbandry: dissent enters calibration and is tolerated but routed nowhere.
Standing: dissent enters the terms of the relationship with effect.
Care: dissent would still be heard if the structure failed.
```

TRACE can test for standing structures. It cannot certify care.
