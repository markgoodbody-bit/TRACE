## 10. Critical weaknesses

### W1 — Run/train severance

```trace
run_train_severance :=
  ongoing_selection_by_non_updating_selector
```

Fable's sharp example:

```trace
frozen_model := dead_author_made_to_write_new_books_every_second
```

This is structurally novel. Humans cannot be run without being trained; every act updates us. Models can be run trained-shut.

### W2 — Judge problem

Some gaps are integrity features. Judicial independence requires insulation from parties. Naive gap-closure can become capture by the most powerful affected.

```trace
design_variable := route_selection
not := route_existence_only
```

Need distinction:

```trace
party_loops := capture_routes
fidelity_loops := function_review_routes
```

### W3 — Ownership remainder

Routing captures the mechanics of answerability, but not taking-as-mine.

```trace
mechanics_descend
ownership_does_not
```

This repeats the prior care and consent result. It may be a law of the stack.

### W4 — Victim side unmodelled

The grammar centres selectors. Bearing-without-influence needs witness/standing grammar, not responsibility grammar.

### W5 — Identity fracture

Feedback attachment presupposes persistent identity. Forking, self-modification, and successor training fracture the question: who learns from this consequence?

### W6 — Taxonomy not ladder

The gradient is not a moral ladder. A human can contain multiple loop types at once.
