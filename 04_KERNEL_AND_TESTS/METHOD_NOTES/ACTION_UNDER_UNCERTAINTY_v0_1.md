# Action Under Uncertainty v0.1

Status: method note. Not validation. Not proof. Not active TRACE spine. Not Kernel v0.3.

Purpose: record the middle-out / coin-principle insight as a disciplined method constraint for TRACE and Mechanical Ethics work.

This note does not add a new operator to the active spine.

---

## 1. Starting Point

Every real chooser begins mid-stream.

```trace
chooser_condition :=
  incomplete_information
  + limited_memory
  + limited_time
  + limited_energy
  + action_required
```

This is not a temporary defect. It is a structural condition.

If a method requires complete knowledge before action, it will fail in ordinary life and fail harder under pressure.

---

## 2. Middle-Out Consequence

Middle-out reasoning does not begin from perfect foundations and does not command the whole system from above.

It begins in the lived middle of an unfolding situation.

Therefore:

```trace
middle_out_reasoning
  -> uncertainty_always_present
  -> error_margin_always_present
```

The method must therefore handle uncertainty as a permanent condition, not as an exception.

---

## 3. Core Rule

Use the simplest model that lets action proceed.

Do not pretend the model is complete.

```trace
act_under_uncertainty :=
  choose_usable_model
  + mark_limits
  + act_boundedly
  + preserve_correction
```

This is neither perfectionism nor recklessness.

It is disciplined action under incomplete knowledge.

---

## 4. Coin Principle

A coin flip is not physically perfect 50/50. Real coins have asymmetries, force vectors, air conditions, surfaces, and hidden variables.

But for many ordinary decisions, treating the coin as 50/50 is sufficient.

The point is not literal truth. The point is usable compression.

```trace
coin_principle :=
  complex_reality
  -> useful_binary_or_simple_model
  -> action
  while:
    simplification_visible_as_simplification
```

The model is a tool.

The model is not the world.

---

## 5. Two Kinds of Uncertainty

Not all uncertainty has the same meaning.

| Type | Meaning | Action posture |
|---|---|---|
| Background uncertainty | ordinary incompleteness that cannot be removed | act through it with visible limits |
| Critical uncertainty | unknown where error may cause hardening, capture, or irreversible harm | slow down, seek evidence, preserve reversibility |

Compression:

```trace
if uncertainty == background:
  act_with_visible_limits

if uncertainty == critical:
  increase_evidence_requirement
  + prefer_reversible_action
  + preserve_correction_path
```

---

## 6. Trust Threshold Is Consequence-Dependent

There is no universal confidence threshold.

The required trust depends on:

- cost of error,
- reversibility,
- number and vulnerability of affected subjects,
- time pressure,
- whether delay itself causes harm.

```trace
trust_threshold :=
  f(error_cost, reversibility, affected_subjects, time_pressure, delay_harm)
```

Working ladder:

| Use | Confidence posture |
|---|---|
| Explore, brainstorm, combine, test | lower confidence acceptable if labelled |
| Draft, structure, make candidate lens | medium confidence acceptable if reversible |
| Promote operator / public claim | high confidence + comparator pressure required |
| Constrain others / safety-critical / irreversible harm | confidence alone insufficient |

Do not demand near-perfect certainty for reversible exploration.

Do not use exploration confidence for irreversible action.

---

## 7. Good-Enough Does Not Mean Unchecked

A good-enough model must still include:

- visible assumptions,
- failure points,
- uncertainty class,
- reversibility check,
- correction path.

```trace
usable_enough :=
  model_sufficient_for_next_step
  + assumptions_visible
  + limits_visible
  + correction_possible
```

If these are absent, "good enough" becomes self-deception.

---

## 8. TRACE Implication

TRACE should be used as a navigation aid under uncertainty, not as a truth engine.

It helps ask:

- What is the current simplified model?
- What has been reduced away?
- Where could the simplification fail?
- Is the uncertainty background or critical?
- Can correction arrive before harm hardens?

```trace
TRACE_use_under_uncertainty :=
  orient
  + test_clocks
  + expose_gates
  + preserve_wrongability
```

---

## 9. Mechanical Ethics Implication

Mechanical Ethics should not be framed as perfect moral knowledge.

It is a human-facing discipline for choosing under pressure while protecting correction, agency, and future-space.

```trace
ME_use_under_uncertainty :=
  better_choice_under_constraint
  not:
    final_moral_certainty
```

---

## 10. Failure Modes

This note fails if used to justify:

- lazy analysis,
- anti-evidence posture,
- premature certainty,
- false binary framing where many options matter,
- acting through critical uncertainty without safeguards,
- ignoring affected subjects,
- claiming uncertainty makes all choices equal.

```trace
demote_if :=
  good_enough == excuse
  OR model_limits_hidden
  OR irreversible_risk_ignored
  OR simplification_confused_with_truth
```

---

## 12. Minimal Use Form

For any case, ask:

1. What model are we using to act?
2. What does it simplify away?
3. Is the remaining uncertainty background or critical?
4. What happens if the model is wrong?
5. Can correction still arrive before hardening?

End.
