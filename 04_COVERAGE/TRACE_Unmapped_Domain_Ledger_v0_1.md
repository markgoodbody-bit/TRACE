# 1. Positive life / flourishing

## Gap

TRACE can detect harm, capture, coercion, and hardening. It does not yet map enough of what makes life worth preserving before the emergency appears.

```trace
positive_life_gap :=
  joy
  + play
  + beauty
  + friendship
  + trust
  + creativity
  + contentment
  + wonder
  + ordinary_belonging
```

## Why it matters

If TRACE only maps catastrophe, it becomes a disaster ethics engine. That is too small. A system must know what it is preserving, not only what it is preventing.

## Candidate carriers

- friendship and loyalty narratives;
- music, comedy, art, dance, sport, craft;
- communities that preserve ordinary life under pressure;
- care practices that produce stable people without capture;
- children's play and safe exploration.

## TRACE questions

```trace
flourishing_preserved_if:
  agency_space_live
  + relation_safe_enough
  + play_possible
  + future_open_enough
  + body_not_under_constant_threat
```

## Falsifier / demoter

If positive life can be handled entirely as low harm plus agency plus future-space, without adding any new operator, then this domain should remain a perspective layer rather than a new TRACE module.

# 2. Markets, money, debt, and pricing

## Gap

TRACE has metric capture, but has not fully mapped how money structures time, access, burden, dependency, and repair.

```trace
market_gap :=
  price_as_gate
  + debt_as_time_capture
  + insurance_as_future_sorting
  + rent_as_access_control
  + profit_as_metric_capture
  + externality_as_hidden_basement
  + asset_ownership_as_future_control
```

## Why it matters

Many systems route harm through ordinary financial mechanisms while appearing voluntary, neutral, or efficient.

## Candidate historical carriers

- enclosure;
- debt peonage;
- company towns;
- 2008 financial crisis;
- subprime lending;
- austerity;
- private equity in care infrastructure;
- insurance exclusion;
- predatory lending and rent extraction.

## TRACE questions

```trace
market_harm_visible_if:
  price_gate_identified
  + dependency_named
  + exit_realism_tested
  + externality_routed_back
  + repair_capacity_not_outbid
```

## Falsifier / demoter

If existing political economy or institutional economics explains the case with equal precision and less new vocabulary, TRACE should translate rather than claim novelty.

# 3. Ecology, evolution, habitat, and life-webs

## Gap

TRACE has touched life, animals, energy, and infrastructure, but has not fully mapped habitats, niches, extinction, symbiosis, predation, domestication, invasive behaviour, and cascade collapse.

```trace
ecology_gap :=
  niche
  + dependency_web
  + predation
  + symbiosis
  + habitat_loss
  + extinction
  + cascade_failure
  + domestication
  + invasive_expansion
```

## Why it matters

Powerful systems do not only act on individual subjects. They alter habitats. AI, markets, states, and infrastructure can change the environment in which agency is possible.

## Candidate carriers

- keystone species;
- pollinator collapse;
- invasive species;
- whale/cetacean standing;
- octopus and alien-mind boundary;
- industrial farming;
- habitat fragmentation;
- climate adaptation limits.

## TRACE questions

```trace
habitat_failure :=
  local_subject_survives
  but:
    niche_destroyed
    + dependency_web_collapses
    + future_space_nonviable
```

## Falsifier / demoter

If ecology remains only metaphor and does not improve actual diagnosis of dependency, habitat, and cascade, it should not be promoted into the core.

# 4. Collective belief, religion, myth, ideology

## Gap

TRACE can name propaganda and reality capture, but has not mapped enough of sacred story, group identity, purity boundaries, heresy detection, martyr logic, conspiracy compression, and belonging capture.

```trace
belief_gap :=
  sacred_story
  + group_identity
  + purity_boundary
  + heresy_detection
  + martyr_logic
  + conspiracy_compression
  + belonging_capture
  + institution_of_meaning
```

## Why it matters

Belief systems bind people. Sometimes they produce care, courage, and restraint. Often they launder hierarchy, obedience, exclusion, and violence. TRACE needs the distinction rather than a flat dismissal.

## Candidate historical carriers

- religious wars;
- cults and coercive belief communities;
- moral panics;
- nationalism;
- conspiracy movements;
- Quaker conscience and low-coercion faith as contrast case;
- liberation theology / care-facing religious practice;
- purity/pogrom logic.

## TRACE questions

```trace
belief_legible_if:
  binding_function_named
  + coercion_level_named
  + dissent_route_checked
  + outgroup_harm_checked
  + care_output_not_assumed
```

## Falsifier / demoter

If belief is reduced to "collective insanity" or "community good" without distinguishing binding, coercion, care, and harm, the TRACE map is failing.

# 6. Body, pain, trauma, illness, addiction, disability

## Gap

TRACE speaks about subjects, harm, and repair, but it can still be too cognitive. Bodies are clocks, limits, records, dependencies, scars, and repair constraints.

```trace
body_gap :=
  pain_signal
  + fatigue
  + trauma_memory
  + addiction_loop
  + medication_dependency
  + disability
  + sleep
  + hunger
  + repair_capacity
```

## Why it matters

A process can look fair while destroying the body-clock of the affected person. Contestability that arrives after exhaustion, relapse, breakdown, pain, or death is not real correction.

## Candidate carriers

- chronic illness and benefits systems;
- addiction recovery and relapse loops;
- trauma-informed care;
- disability access failures;
- medication dependency at gates;
- policing encounters involving mental distress;
- prisons, hospitals, care homes;
- sleep deprivation and coercive interrogation.

## TRACE questions

```trace
body_clock_failure :=
  formal_route_exists
  but:
    pain
    + fatigue
    + dependency
    + trauma
    make_route_unusable_in_time
```

## Falsifier / demoter

If body effects are treated only as evidence of harm, not as constraints on timing, agency, and correction, TRACE is under-reading the case.

# 8. Mathematical spine

## Gap

TRACE is currently mostly structured prose and glyphs. The mathematical layer is implied more than formalised.

```trace
math_gap :=
  state_space
  + transition_function
  + observability
  + control_lever
  + feedback_loop
  + dependency_graph
  + hardening_time
  + correction_window
  + loss_function_capture
```

## Minimal candidate formalisation

```trace
state := S_t
action := a_t
transition := S_t --a_t--> S_t+1
harm_delta := ΔH(S_t, S_t+1)
hardening_time := T_h
correction_arrival := T_c
correction_window := T_h - T_c
TRACE_failure_if := correction_window <= 0
```

## Why it matters

Without a mathematical spine, TRACE risks remaining a structured moral vocabulary. With too much premature formalisation, it risks fake precision. The next maths layer must be light, testable, and demotable.

## Candidate anchors

- control theory;
- graph theory / dependency networks;
- dynamical systems;
- decision theory;
- game theory;
- information theory;
- reliability engineering;
- queueing / latency;
- mechanistic interpretability as actual system reading, not metaphor.

## Falsifier / demoter

If the notation does not improve prediction, diagnosis, comparison, or compression, it is decorative and should be removed.

# Cross-domain control questions

Use these before adding a new domain to core TRACE.

```trace
domain_promotion_requires:
  new_failure_shape
  + real_case_or_source_path
  + operator_candidate
  + inward_falsifier
  + ordinary_comparator
  + no_bootstrap_bloat
```

Questions:

1. What does this domain let TRACE see that it currently misses?
2. What existing field already handles this better?
3. What operator would it add, if any?
4. What would demote it?
5. Which existing cluster does it pressure?
6. What historical case can test it?
7. What is the danger of importing it?
