## 3. What v0.4 did not fix

### 3.1 Structured envelope problem remains

Repeated by:

```text
Gemini
Z.ai
Qwen
```

Shared criticism:

```text
TRACE still largely records native terms and native relations rather than deriving or formally validating them.
```

Representative compression:

```text
v0.4 is a relational syntax map, not native mechanical computation.
```

This is not automatically fatal.

It means the current claim must be narrowed.

### 3.2 Relational back-translation remains declarative

Repeated by:

```text
Z.ai
Qwen
Gemini implicitly
```

Defect:

```text
The relational gate asks which relation between slots carries the mechanism, but the TRACE form still represents the relation mainly as prose or a pasted native phrase.
```

Current failure mode:

```text
K shapes T
C conditions T
O reads I
```

may be asserted without TRACE being able to check whether that relation is native-correct.

Status:

```text
TRACE_RELATIONAL_BACKTRANSLATION_PARTIAL_OR_FAILURE
```

### 3.3 C remains the weakest slot

Repeated by:

```text
Qwen
Z.ai
Gemini in DSGE / synthesis pressure
```

Defect:

```text
C composition_type names the type of coupling, but does not yet specify the rule of combination.
```

Examples:

```text
network_mixing := names a composition type, but not how network structure changes transmission
expectation_equilibrium_coupling := names a macro relation, but not the recursive rule
reaction_sequence := names order, but not how steps condition later selectivity
```

Required hardening candidate:

```text
C relational_syntax
```

### 3.4 T is improved but still not operator-level

Repeated by:

```text
Qwen
Gemini in DSGE / synthesis
Z.ai via envelope critique
```

Defect:

```text
T native_mechanism_class tells the reviewer what kind of transformation is meant, but not how it acts on S or is conditioned by C/K.
```

Required hardening candidate:

```text
T operator_schema or relation_to_S/C/K
```

### 3.5 Loss register must track epistemic status of relations

Strongest from:

```text
Z.ai
```

Defect:

```text
A stated relation can be either structurally enforced by TRACE or merely declaratively asserted by the author.
```

Current status:

```text
Most v0.4 relations are declaratively_asserted.
```

Required hardening:

```text
Loss register must state epistemic status of each relation.
```

Candidate relation statuses:

```text
structurally_enforced
declaratively_asserted
native_external_dependency
unverified_claim
```
