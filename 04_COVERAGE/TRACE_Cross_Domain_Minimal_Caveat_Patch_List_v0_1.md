## 1. Patch discipline

Do not patch all parent files automatically.

Apply a caveat only if one of the following occurs:

```text
1 external domain-aware reviewer confirms the issue
2 independent review repeats the same concern
3 the parent file clearly overclaims even without expert input
4 Mark explicitly chooses consolidation mode
```

Patch target should be minimal:

```text

native clock
native constraint
native failure mode
native correction/control channel
ethics/governance boundary
fail condition
```

Do not expand TRACE theory while applying caveats.

### 2.1 Quantum information — error correction / no-cloning caveat

Dry-run issue:

```text
TRACE may describe fragile state loss too generally and lose the formal content of quantum state, measurement, no-cloning, and fault-tolerance thresholds.
```

Candidate patch target:

```text
native constraint
```

Candidate caveat:

```text
Quantum error correction does not clone unknown quantum states. It protects logical information by encoding it across entangled physical degrees of freedom under threshold assumptions and syndrome/error-correction procedures.
```

Reason:

```text
This keeps the TRACE translation from becoming generic redundancy / backup language.
```
