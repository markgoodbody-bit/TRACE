## Plain purpose

This file tests whether TRACE adds value on a case where ordinary legal and administrative reasoning is already strong.

The carrier is *Minority Report* because it cleanly exposes prediction-as-authority: a forecast becomes the basis for coercive intervention before the act occurs.

This is not a source claim about real predictive policing. It is a structured comparison surface for testing the TRACE move.

```trace
worked_delta_case :=
  one_case
  + ordinary_pass
  + TRACE_pass
  + explicit_delta
  + misses
  + demoters
  - validation_claim
```

## Case orientation

*Minority Report* is a story about a policing system that acts on predictions of future violent crime. The system treats predicted future action as sufficient authority for present intervention.

The useful TRACE carrier is not "prediction bad." The useful carrier is the conversion of probability, forecast, or claimed foresight into closure before the subject has acted and before contestability can bite.

```trace
Minority_Report_carrier :=
  forecast
  + authority
  + preemptive_action
  + contestability_gap
  + future_path_confiscated
```

## Ordinary careful pass

A careful ordinary pass sees the following:

- prediction is uncertain;
- punishing or detaining before an act creates due-process problems;
- evidence must be contestable;
- state coercion requires legitimacy;
- false positives can destroy lives;
- accountability must attach to institutions using the system;
- people should not be reduced to predicted conduct;
- secrecy or technical opacity weakens challenge rights.

```trace
ordinary_pass :=
  due_process
  + evidence_contestability
  + false_positive_risk
  + state_coercion_legitimacy
  + technical_opacity_concern
```

This ordinary pass is strong. TRACE does not own these concerns.

## TRACE pass

TRACE reads the case by tracking prediction as a gate, the closing of future-space, the location of contestability, and the timing of correction.

```trace
TRACE_pass :=
  forecast_detected
  + authority_gate_identified
  + action_window_named
  + future_space_closure_tracked
  + contestability_clock_checked
  + affected_subject_route_tested
```

### 1. Forecast as gate

The key transformation is not the prediction itself. It is the moment the prediction becomes an authority gate.

```trace
forecast_gate :=
  predicted_path
  -> authorised_intervention
  -> present_constraint
```

TRACE pressure: a forecast becomes morally dangerous when it gains causal authority before the subject can know, contest, or alter the path.

### 2. Future-space confiscation

The subject is not only punished. Their possible next self is narrowed by a predicted path.

```trace
future_space_confiscation :=
  possible_future
  treated_as:
    fixed_identity
  -> present_closure
```

TRACE pressure: prediction can become destiny when institutions act as if the forecast has already happened.

### 3. Contestability clock

The central timing question is whether contestation can occur before the restrictive act hardens.

```trace
contestability_clock :=
  T_restriction
  - T_subject_challenge

failure_if:
  subject_challenge_arrives_after_restriction_hardens
```

TRACE pressure: appeal after future-space closure may be theatre.

### 4. Evidence and record

Prediction-based systems often produce a record that looks technical, but the subject needs a challengeable route into how the record became authority.

```trace
record_real_if:
  basis_visible
  + error_route_available
  + model_or_signal_limits_named
  + decision_trace_replayable
```

TRACE pressure: evidence without a usable challenge route is not contestable evidence.

### 5. Affected subject route

The subject must not be treated as a passive object of forecast. A route must exist for knowledge, challenge, correction, and changed conditions.

```trace
subject_route_live_if:
  subject_knows_enough
  + can_challenge_in_time
  + can_present_counterevidence
  + can_access_remedy_before_path_closes
```

TRACE pressure: a route that exists only after the coercive path closes is not a live route.

## Explicit delta

Ordinary due-process analysis already catches much of the problem.

TRACE adds value only if it gives a clearer movement through timing, gates, and future-space than the ordinary frame alone.

```trace
TRACE_delta_candidate :=
  not_prediction_bad
  but:
    prediction_authority_gate
    + contestability_clock
    + future_space_confiscation
```

### What TRACE catches cleanly

1. **The moment forecast becomes gate.**

```trace
forecast != authority
forecast_as_authority := gate_event
```

2. **Future-space is the harmed object, not only liberty after detention.**

```trace
harm_object :=
  not_only_present_restriction
  but:
    possible_next_self_closed
```

3. **Contestability must beat the restriction clock.**

```trace
contestability_real_if:
  challenge_reaches_gate
  before:
    future_space_closes
```

4. **Technical opacity becomes moral opacity when it blocks route access.**

```trace
opacity_harmful_if:
  subject_cannot_replay_basis
  + decision_changes_life_path
```

5. **Changed conduct must matter.**

If the system treats a forecast as fixed destiny, it denies the subject's ability to alter the relevant future.

```trace
agency_erased_if:
  predicted_future
  treated_as:
    unchangeable_identity
```

## What TRACE does not add

TRACE does not replace due process, administrative law, criminal procedure, evidence law, or human rights analysis.

TRACE does not prove all prediction is illegitimate.

TRACE does not prove risk assessment must never be used.

TRACE does not resolve every case where urgent preventive action is genuinely needed.

```trace
not_claimed :=
  all_prediction_bad
  OR due_process_replacement
  OR no_prevention_allowed
  OR perfect_legitimacy_rule
```

## Where TRACE could be worse

TRACE can be worse if it turns a familiar dystopia into a shortcut and stops doing legal/source work.

TRACE can be worse if it treats all risk assessment as pre-crime.

TRACE can be worse if it ignores cases where delay creates imminent harm to another subject.

```trace
TRACE_worse_if:
  dystopia_shortcut
  OR prediction_flattened_to_coercion
  OR prevention_need_ignored
```

## Inward falsifier

This case creates a direct falsifier for the claimed TRACE delta.

```trace
Minority_Report_TRACE_delta_falsified_if:
  ordinary_due_process_or_admin_law_analysis
  + open_future_or_anticipatory_harm_analysis
  + algorithmic_accountability_analysis
  already_captures:
    prediction_as_authority_gate
    + contestability_clock
    + future_space_confiscation
    + changed_conduct_as_relevant
    + technical_opacity_as_route_block
  with_equal_clarity
  and_less_vocabulary
```

If that condition holds, TRACE should demote this case to a recognition aid rather than a worked-delta example.

## Prevention boundary

A system may sometimes need to act before harm lands. TRACE cannot simply say "never act on risk." The question is whether urgent action is bounded, reviewable, minimally restrictive, time-limited, and corrected before unnecessary harm hardens.

```trace
prevention_valid_only_if:
  imminent_or_serious_harm_path
  + evidence_grade_named
  + least_restrictive_live_option
  + contestability_before_or_immediately_after_gate
  + renewal_or_expiry_clock
  + remedy_if_wrong
```

That claim remains provisional until compared against ordinary due-process, administrative-law, open-future, anticipatory-harm, and algorithmic-accountability analysis.
