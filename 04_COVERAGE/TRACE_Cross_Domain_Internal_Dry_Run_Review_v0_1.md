## 2. Strongest thing TRACE preserves

The strongest preserved structure is:

```text
state -> transition -> constraint -> clock -> failure/hardening -> correction/control
```

This is not merely decorative.

Across the eight test domains, the map repeatedly identifies:

```text
what is changing
what constrains the change
what clock matters
what failure looks like
what channel can still intervene
where analogy stops
```

That is a real bridge-language candidate.

## 3. Biggest flattening / overreach risk

The biggest risk is that TRACE's primitives become too broad:

```text
state
constraint
clock
failure
correction
```

These can become generic systems-language if not anchored to the native mechanism.

The danger line is:

```text
TRACE notices a structural rhyme
then forgets the field-specific machine that makes the rhyme meaningful
```

## 4. Native mechanism most at risk

Highest-risk domains:

```text
quantum information
algebraic topology
generative syntax
organic synthesis
```

Reason:

```text
Their native mechanisms are formal and technical enough that loose analogy can sound plausible while losing what actually matters.
```

Specific risks:

```text
quantum: treating decoherence as generic information loss without preserving the formal role of state, measurement, and error thresholds
algebraic topology: treating invariance as ordinary continuity rather than exact mathematical invariant under allowed maps
syntax: treating phase/access as institutional closure rather than formal derivational constraint
organic synthesis: treating protecting groups as generic protection rather than reaction-selectivity strategy
```
