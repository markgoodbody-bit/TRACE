## Plain purpose

This file tests the remaining open question from the Prediction Authority comparator: whether TRACE's `future_space_confiscation` operator adds cross-domain value beyond existing open-future and anticipatory-harm language.

This is a source-thin run. It uses accessible summaries and abstracts, not a full literature review.


## Claim being tested

```trace
future_space_confiscation :=
```


The question is whether these are genuinely useful TRACE compressions, or just a relabeling of existing open-future and anticipatory-harm ideas.

This family already captures the idea that present decisions can wrong a person by closing future options before the later person can choose. That is close to TRACE's future-space language.

## Component test

### A. Does open-future language already capture future-space confiscation?

Partly yes.

Open-future language captures premature closure of possible later lives. TRACE should not claim originality over this component.

```trace
future_space_confiscation_component :=
  captured_locally_by_open_future_family
```

TRACE residual:

TRACE links open-future closure to gate, clock, contestability, and correction-before-hardening.

```trace
TRACE_residual :=
  open_future
  routed_through:
    gate
    + clock
    + correction_window
    + affected_subject_route
```

Assessment:

```trace
future_space_confiscation :=
  captured_locally
  + compressed_by_TRACE
  + transferred_by_TRACE
```

### B. Does anticipatory ethics already capture future harm before it arrives?

Partly yes.

Anticipatory ethics already asks how to reason under uncertainty about future consequences. TRACE should not claim to invent future-facing ethical reasoning.

```trace
anticipatory_harm_component :=
  captured_locally_by_anticipatory_ethics
```

TRACE residual:

TRACE is narrower and more mechanical. It asks whether a subject-facing correction route can still reach the causal path before hardening.

```trace
TRACE_residual :=
  not_general_future_ethics
  but:
    correction_before_hardening
    + gate_reachability
    + route_realism
```

Assessment:

```trace
anticipatory_harm :=
  captured_locally
  + narrowed_by_TRACE_to_correction_window
```

### C. Does scored-society work already capture predictive closure?

Substantially yes.

Scored-society and algorithmic-accountability work already captures scoring, opacity, social participation effects, due-process pressure, and the need for public review or challenge.

```trace
predictive_closure_component :=
  strongly_captured_by_scored_society_family
```

TRACE residual:

TRACE may help transfer this structure beyond credit/scoring and into general prediction-authority systems.


Assessment:

```trace
  high_local_capture

```

## Narrowed operator

The safer operator is not broad `future_space_confiscation` alone.

```trace
safer_operator :=
  future_space_closure_at_gate
```

Definition:

```trace
future_space_closure_at_gate :=
  prediction_or_classification
  gains_authority
  before:
    affected_subject_can_know_contest_or_alter_path
  causing:
    available_future_paths_to_shrink
```

This operator preserves the open-future insight while adding TRACE's timing/gate/correction structure.
