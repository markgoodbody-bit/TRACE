## 3. Boundary-language ambiguity

The current example uses boundary language in two senses:

1. boundary of a topological space, as when the disk has a boundary circle;
2. boundary in the chain-complex sense, where a cycle may or may not be the boundary of a higher-dimensional chain.

This is dangerous because TRACE already uses `boundary` as a general role.

```trace
boundary_ambiguity :=
  topological_boundary
  vs
  chain_boundary
  vs
  TRACE_boundary_role
```

Patch required:

```trace
patch_boundary_terms :=
  distinguish:
    topological_boundary_of_space
    + chain_boundary_under_boundary_operator
    + TRACE_B_Top_boundary_role
```

Failure if unpatched:

```trace
failure_if_unpatched :=
  boundary_as_generic_edge_returns
  + cycle_boundary_distinction_blurs
```

## 5. Circle claim attack

The example says `S1` has one independent 1-cycle and is not filled inside `S1`.

Useful: this captures the core contrast.

Weakness: without explicit chain-complex context, `not filled inside S1` can sound like a visual statement rather than a homological statement.

```trace
attack_circle_claim :=
  useful_as_intuition
  but:
    needs_chain_boundary_context
    + must_not_reduce_to_visual_hole
```

Patch:

```trace
patch_circle_claim :=
  express_as:
    relevant_1_cycle_not_boundary_in_simplified_homology_probe
```

## 6. Disk claim attack

The example says the disk has a boundary circle and the loop bounds a filled 2-dimensional region.

Useful: this distinguishes filled disk from circle.

Weakness: topological boundary of the disk and chain boundary can be confused.

```trace
attack_disk_claim :=
  useful_for_filled_region_contrast
  but:
    boundary_circle_exists_may_confuse_topological_boundary_with_chain_boundary
```

Patch:

```trace
patch_disk_claim :=
  state:
    D2_contains_filling_region_for_boundary_loop_in_simplified_probe
    + topological_boundary_and_chain_boundary_are_not_identical_terms
```
