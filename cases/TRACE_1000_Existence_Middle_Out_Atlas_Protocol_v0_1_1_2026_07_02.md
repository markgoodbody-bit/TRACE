## 6. Designation-by-omission flag

Use `D_OMISSION_FLAG` when a scope may be affected but is absent from the designation lattice.

```trace
D_OMISSION_FLAG :=
  possible_scope_absent_from_frame
  + C_may_proceed_without_D_record
  + omission_itself_requires_record
```

Boundary:

```trace
not_named != not_affected
not_designated != unprotected
unimagined != unreal
```

## 7. Standing / husbandry / proxy-standing discipline

Direct standing test:

```trace
standing_test := can_the_managed_party_alter_the_management_frame?
```

But this presupposes a present expressive party. For absent, future, voiceless, unknown, dead, simulated, or not-yet-visible parties, use proxy standing.

```trace
proxy_standing :=
  proxy_channel_for_absent_or_voiceless_or_unknown_scope
  + forced_routing_to_constraint_or_designation_layer
  + proxy_contamination_record
  + expiry_or_recheck
  - care_certification
  - final_designation
```

Proxy standing ladder:

```trace
proxy_standing_status :=
  no_proxy
  | actor_controlled_proxy
  | independent_proxy_no_write_access
  | independent_proxy_with_forced_routing
  | plural_proxy_with_conflict_record
  | proxy_plus_later_recontact_or_reversal_path
  | unknown
```

Boundary:

```trace
proxy_claim != affected_party_claim
proxy_channel != standing_certification
absence_is_the_husbandry_default
```

## 10. Entity-handshake and handshake-limit discipline

The entity-handshake requires mutual aperture, response capacity, and correction handles.

```trace
handshake_requires := mutual_aperture + response_capacity + correction_handle
```

When these fail, mark:

```trace
HANDSHAKE_LIMIT_FLAG :=
  no_shared_aperture
  | one_way_aperture
  | inverting_aperture
  | no_response_capacity
  | none
```

Boundary:

```trace
one_sided_handshake := husbandry_risk
handshake_is_peer_protocol
non_peer_case_requires_unilateral_restraint_plus_proxy_or_caution
```
