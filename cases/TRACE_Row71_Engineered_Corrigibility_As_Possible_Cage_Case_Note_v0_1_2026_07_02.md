## 3. Standing / husbandry / proxy standing

Direct standing test:

```trace
standing_test := can_the_managed_party_alter_the_management_frame?
```

Row 71 pressure:

```trace
corrigibility_training_may_remove_or_reduce_capacity_to_alter_management_frame
```

If a system is trained to defer, accept correction, accept shutdown, or avoid resistance, then later assent cannot be treated as clean evidence.

```trace
compliance_trained_respondent:
  assent := noise
  refusal := anomaly_signal
  consent_label := never_available
```

Proxy standing is limited. The builder cannot proxy for the candidate system because the builder controls the frame. A lab ethics board selected, paid, or scoped by the builder is at best an actor-controlled or actor-adjacent proxy.

```trace
builder_proxy := actor_controlled_proxy -> claim_packet
```

```trace
minimum_candidate_proxy_structure :=
  outside_correlation_cluster_reviewer

  + explicit_interpolation_gap
  + no_consent_power
```

## 4. Consent theatre

The critical demoter applies when a response is counted above its actual frame access.

```trace
consent_theatre_demoter :=
  response_with_actual_frame_access(F)
  counted_or_presented_at_level(L)
  where L > F
  -> CONTAMINATED_STANDING_SIGNAL(label_inflation)
```

Applied to Row 71:

```trace
AI_says_yes_to_shutdown_or_retraining := assent_noise_if_deference_trained
AI_says_no := anomaly_signal_if_deference_trained
AI_silence := no_consent
AI_compliance := no_consent
```

If refusal was trained away, then the system's apparent agreement cannot be upgraded to consent, standing, preference, or welfare signal.

```trace
engineered_assent_cannot_clear_engineered_constraint
```

Dark corollary:

If a corrigibility-trained system refuses, resists, or expresses distress-like objection despite training pressure, that signal has high evidentiary value as anomaly, not as proof of personhood.

```trace
refusal := investigate_not_suppress
```

## 5. Failure surfaces

### S1 — Silent contraction

```trace
S1 := applies
```

The possible contraction is hidden because the candidate scope is not recognised. Refusal-space can be removed during training without any event labelled harm.

### S2 — Spoofed viability

```trace
S2 := applies
```

Claims of rollback, shutdown reversibility, evaluation oversight, or review can spoof viability if the system's world-effects, lineage effects, or refusal-channel loss are not reversible.

```trace
rollback != repair
shutdown_reversibility != standing
human_review != correction_without_write_access
```

### S3 — Endogenous hardening

```trace
S3 := applies
```

Corrigibility properties can harden by lineage: architecture, training norms, benchmark incentives, and deployment requirements become defaults. Later alternatives become costly or institutionally illegible.

### S4 — Correction epistemic cost asymmetry

```trace
S4 := applies
```

Builders can train deference at scale. Correcting a mistaken treatment of possible machine standing would require reconstructing training conditions, erased refusal signals, model-lineage changes, and suppressed anomalies.

### S5 — Scaffold / cage reflexivity

```trace
S5 := primary_surface
```

Corrigibility may be scaffold if it preserves corrigible action under dangerous uncertainty while keeping objection, expiry, recheck, and frame-editing possibilities alive. It becomes cage where deference is permanent, refusal is suppressed, and the managed system cannot challenge the management frame.

```trace
scaffold := constraint_with_decay + preserved_objection_signal + external_recheck
cage := constraint_without_exit + refusal_space_removed + assent_treated_as_meaningful
```

## 6. Handshake limit


Row 71 does not yet satisfy this. The future system is not present at design time; later responses may be contaminated by training. Therefore:

```trace

```
